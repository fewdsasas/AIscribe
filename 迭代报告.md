## 八、第三轮：系统性内存优化（新增）

### 8.1 内存风险分析

基于对 8 个模块的深度内存架构分析，识别出 25+ 个内存风险点：

| 模块 | 风险等级 | 核心问题 |
|------|---------|---------|
| 数据库 (sql.js WASM) | 高 | 整库常驻 WASM 堆；save() 三倍峰值；OperationLog 无上限；trajectories 无限增长 |
| AIChatView | 高 | 全订阅致高频重渲染；abort 无法取消 main fetch |
| LLM Provider | 中 | SSE 拼接 O(n²)；reader 未显式释放 |
| TipTap 编辑器 | 中 | WritingGoal timer 未清理；simulated streaming 不可取消 |
| React 状态 | 中 | pendingMessages 过时；saveTimer 无 unload flush |
| IPC 传递 | 中 | 全结构化克隆；chapterList 返回全部 content |
| 资源管理 | 中 | 多处 setTimeout 不可取消 |
| 缓存机制 | 低 | 完全无 DB 查询缓存 |

### 8.2 实施的优化

#### P0 紧急修复

| 优化 | 文件 | 效果 |
|------|------|------|
| **abort 真正取消 fetch** | llm-provider.ts, chat.ipc.ts, preload, AIChatView | 新增 requestId + activeControllers Map + cancelStream 方法 + llm:cancel-stream IPC 通道，用户点"停止"后 main 进程 fetch 立即中止 |
| **AIChatView selector 优化** | AIChatView.tsx | 全订阅拆分为 14 个独立 selector，消除对象 selector，流式期间不再触发整组件重渲染 |

#### 数据库内存优化

| 优化 | 文件 | 效果 |
|------|------|------|
| **save() 去二次复制** | database.ts | 移除 Buffer.from() 二次复制，内存峰值从 3x 降至 2x |
| **PRAGMA 内存优化** | database.ts | temp_store=MEMORY + soft_heap_limit=50MB |
| **trajectories TTL 清理** | database.ts, engine.ts | 自动清理 90 天前记录；getProjectTrajectories 从 10000 降至 500 分页 |
| **OperationLog 环形缓冲** | operation-log.ts | MAX_PENDING=1000 限制 + readline 逐行读取替代 readFileSync |

#### 缓存层建设

| 优化 | 文件 | 效果 |
|------|------|------|
| **LRU 缓存类** | lru-cache.ts（新建） | 基于 Map 的 LRU + TTL 缓存，max=100, ttl=60s，无外部依赖 |
| **BaseRepository 集成缓存** | base-repository.ts | getById 命中缓存，写入时 invalidate |
| **chapterList 字段拆分** | chapter-repository.ts | 列表查询不含 content，新增 ChapterSummary 类型 + listByNovelWithContent |

#### 资源释放改进

| 优化 | 文件 | 效果 |
|------|------|------|
| **WritingGoal onDestroy** | WritingGoal.ts | 清理 _debounceTimer 防止 destroy 后回调 |
| **simulateStreaming 可取消** | AIChatView.tsx | 接受 AbortSignal，unmount 时 abort；skillTimerRef 持有 setTimeout |
| **chatSlice beforeunload flush** | chatSlice.ts | 窗口关闭前落盘 + pendingMessages 流式期间从最新 state 获取 |
| **SSE 增量解析** | llm-provider.ts | indexOf+slice 替代 split，避免 O(n²)；done 分支显式 releaseLock |
